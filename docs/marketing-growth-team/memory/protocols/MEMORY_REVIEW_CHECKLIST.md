# Memory Review Checklist

Use this checklist before promoting an item from `REVIEW_QUEUE.md`.

## Approve If

- The item will likely improve future work.
- It has a source, metric, or stakeholder approval.
- It is not a secret, private datum, or raw transcript.
- It has a clear scope.
- It has confidence and a review date.
- It belongs in the target file.

## Reject Or Revise If

- It is a one-off task detail.
- It is unsupported or speculative.
- It conflicts with approved memory and no decision was made.
- It is too broad to act on.
- It belongs as a Skill, not memory.

## Promotion Targets

- Brand, compliance, audience, offer, campaign, or trusted-source facts go to `memory/shared`.
- Cross-agent decisions and operating rules go to `memory/orchestrator`.
- Role-specific reusable patterns go to `memory/agents`.
- Repeatable procedures go to `memory/orchestrator/SKILL_BACKLOG.md`.
