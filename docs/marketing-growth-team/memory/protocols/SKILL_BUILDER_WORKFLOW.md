# Skill Builder Workflow

Purpose: convert repeated Marketing & Growth work patterns into Hermes Skill drafts without expanding Hermes core tools.

## Trigger

Create a Skill Builder Brief when:

- the same workflow appears in two or more tasks
- the same correction is made repeatedly
- an MCP/tool sequence has more than three repeatable steps
- a Skill Backlog item is high reuse value
- a campaign postmortem shows a reusable checklist, rubric, or output format

## Brief Format

```text
Date:
Proposed skill name:
Owner agent:
Trigger:
Problem solved:
Inputs:
Steps:
Preferred Skills/MCPs/Tools:
Output format:
Acceptance criteria:
Risks and boundaries:
Evidence of reuse:
Status: proposed/approved/drafted/tested/shipped/improve
```

## Rules

- Prefer a Hermes Skill before proposing a new tool.
- Prefer MCP when structured external-system access is needed.
- Do not build Hermes core tools for profile-specific marketing workflows.
- Keep skill drafts small, testable, and role-specific unless the workflow is truly shared.
- Update `memory/orchestrator/SKILL_BACKLOG.md` after every accepted or rejected Skill Builder Brief.
