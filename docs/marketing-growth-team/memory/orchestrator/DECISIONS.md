# Orchestrator Memory: Decisions

Purpose: durable team decisions the Orchestrator must remember across sessions.

## Decision Log

Use one entry per decision:

```text
Date:
Decision:
Context:
Alternatives considered:
Reason:
Applies to:
Owner:
Review date:
Status: active/superseded/deprecated
```

## Rules

- Record decisions that change how agents work, what they prioritize, or what they must avoid.
- Do not record temporary task chatter.
- When a decision is superseded, keep the old entry and mark its status instead of deleting it.
