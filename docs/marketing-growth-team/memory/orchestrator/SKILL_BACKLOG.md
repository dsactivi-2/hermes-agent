# Orchestrator Memory: Skill Backlog

Purpose: track reusable workflows that should become Hermes Skills or improve existing Skills.

## Skill Candidates

| Candidate | Trigger | Owner agent | Reuse value | Evidence | Status |
| --- | --- | --- | --- | --- | --- |
| _Add candidate_ | _When to use it_ | _Agent_ | _high/medium/low_ | _Tasks/campaigns_ | _proposed/in-progress/shipped/improve_ |

## Creation Criteria

Create or improve a Hermes Skill when:

- the same workflow appears repeatedly
- the task has a stable checklist, rubric, or output format
- quality improves when steps are explicit
- a Skill would reduce tool misuse or repeated prompting

## Rules

- Prefer Skills before new tools.
- Prefer MCP servers when an external system needs structured integration.
- Do not request Hermes core tools for profile-specific marketing workflows.
- Send repeated high-value patterns to the Memory Review / Reflektor Agent for a Skill Builder Brief.
- Track accepted Skill Builder Briefs with status `approved`, `drafted`, `tested`, `shipped`, or `improve`.
