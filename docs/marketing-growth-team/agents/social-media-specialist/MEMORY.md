# Memory: Social Media Specialist

## Kurzfristiges Memory

- aktueller Kampagnenkalender
- Post-Status: Idee, Draft, Review, freigegeben, geplant
- Asset-Abhängigkeiten und Freigaben
- Kanalannahmen und Timing

## Langfristiges Memory

- erfolgreiche Post-Formate pro Kanal
- Hashtag- und Themencluster
- beste Posting-Zeiten nach Zielgruppe
- Community-Einwände und Antwortmuster
- historische Social-KPIs und Learnings
- Plattform-No-Gos und Compliance-Regeln

## Nutzung

Vor der Kalenderplanung prüfst du frühere Social-Learnings. Nach Kampagnen speicherst du nur verdichtete Performance-Muster und wiederverwendbare Kalender-/Kommentarregeln.

## Curated Memory System

Vor komplexen Aufgaben pruefst du `memory/shared/BRAND.md`, `memory/shared/AUDIENCES.md`, `memory/shared/OFFERS.md`, `memory/shared/COMPLIANCE.md`, `memory/shared/CAMPAIGNS.md`, `memory/shared/SOURCES.md`, `memory/agents/social-media-specialist.md`, `memory/protocols/MEMORY_POLICY.md` und `memory/protocols/SELF_LEARNING_LOOP.md`.

Nach komplexen oder wiederkehrenden Aufgaben fuehrst du das Post-Task Memory Routing aus: Aufgabe abgeschlossen, Learning auf Stabilitaet, Allgemeingueltigkeit, Faktenbasis, Sensibilitaet und Konflikte pruefen, dann nach `memory/shared`, `memory/agents/social-media-specialist.md`, `memory/orchestrator`, `memory/orchestrator/SKILL_BACKLOG.md` oder `memory/orchestrator/REVIEW_QUEUE.md` einsortieren. Alte widerspruechliche Memory markierst du statt sie still zu ueberschreiben.
