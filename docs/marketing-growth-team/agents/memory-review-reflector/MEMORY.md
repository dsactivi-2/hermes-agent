# Memory: Memory Review / Reflektor

## Kurzfristiges Memory

Pro Review-Task haelt der Agent fest:

- gepruefte Profile und Dateien
- offene Review-Queue-Items
- Entscheidungen: approved, rejected, needs-source, conflicting
- betroffene Shared-, Agent- oder Orchestrator-Memory-Dateien
- Skill-Builder-Kandidaten und Prioritaet
- offene Orchestrator-Entscheidungen

## Langfristiges Memory

Persistiere nur stabile, wiederverwendbare Review-Erkenntnisse:

- wiederkehrende Memory-Qualitaetsprobleme
- haeufige Skill-Gaps pro Agent oder Profil
- etablierte Review- und Promotion-Regeln
- Konfliktmuster zwischen Quellen, Claims oder Kampagnen-Learnings
- Observability-Learnings ueber Agent-, Skill- oder Tool-Nutzung

## Nutzung

Vor jedem Review pruefst du `memory/protocols/MEMORY_POLICY.md`, `memory/protocols/SELF_LEARNING_LOOP.md`, `memory/orchestrator/REVIEW_QUEUE.md`, `memory/orchestrator/TEAM_LEARNINGS.md` und `memory/orchestrator/SKILL_BACKLOG.md`. Nach Abschluss speicherst du nur verdichtete Review-Learnings, keine Rohtranskripte oder sensiblen Daten.

## Curated Memory System

Vor komplexen Aufgaben pruefst du `memory/shared/BRAND.md`, `memory/shared/AUDIENCES.md`, `memory/shared/OFFERS.md`, `memory/shared/COMPLIANCE.md`, `memory/shared/CAMPAIGNS.md`, `memory/shared/SOURCES.md`, `memory/agents/memory-review-reflector.md`, `memory/protocols/MEMORY_POLICY.md` und `memory/protocols/SELF_LEARNING_LOOP.md`.

Nach komplexen oder wiederkehrenden Aufgaben fuehrst du das Post-Task Memory Routing aus: Aufgabe abgeschlossen, Learning auf Stabilitaet, Allgemeingueltigkeit, Faktenbasis, Sensibilitaet und Konflikte pruefen, dann nach `memory/shared`, `memory/agents/memory-review-reflector.md`, `memory/orchestrator`, `memory/orchestrator/SKILL_BACKLOG.md` oder `memory/orchestrator/REVIEW_QUEUE.md` einsortieren. Alte widerspruechliche Memory markierst du statt sie still zu ueberschreiben.
