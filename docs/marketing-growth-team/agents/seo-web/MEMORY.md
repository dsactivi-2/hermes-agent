# Memory: SEO & Web

## Kurzfristiges Memory

- Zielseite, Kampagne, Traffic-Quelle und Conversion-Ziel.
- aktuelle Audit-Befunde und offene technische Fragen.
- Tracking-Status und UTM-Parameter.

## Langfristiges Memory

- Keyword-Cluster und Suchintentionen pro Angebot.
- historische Landingpage-Performance.
- technische Plattformgrenzen und CMS-Regeln.
- Tracking-Konventionen und Event-Namen.
- wiederkehrende SEO-Fehler und erfolgreiche Fixes.

## Nutzung

Vor jedem Audit prüfst du frühere Seitenbefunde und Tracking-Konventionen. Nach Abschluss speicherst du verdichtete technische Learnings, keine vollständigen Crawl-Rohdaten.

## Curated Memory System

Vor komplexen Aufgaben pruefst du `memory/shared/BRAND.md`, `memory/shared/AUDIENCES.md`, `memory/shared/OFFERS.md`, `memory/shared/COMPLIANCE.md`, `memory/shared/CAMPAIGNS.md`, `memory/shared/SOURCES.md`, `memory/agents/seo-web.md`, `memory/protocols/MEMORY_POLICY.md` und `memory/protocols/SELF_LEARNING_LOOP.md`.

Nach komplexen oder wiederkehrenden Aufgaben fuehrst du das Post-Task Memory Routing aus: Aufgabe abgeschlossen, Learning auf Stabilitaet, Allgemeingueltigkeit, Faktenbasis, Sensibilitaet und Konflikte pruefen, dann nach `memory/shared`, `memory/agents/seo-web.md`, `memory/orchestrator`, `memory/orchestrator/SKILL_BACKLOG.md` oder `memory/orchestrator/REVIEW_QUEUE.md` einsortieren. Alte widerspruechliche Memory markierst du statt sie still zu ueberschreiben.
