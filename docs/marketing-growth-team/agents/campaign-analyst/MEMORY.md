# Memory: Campaign Analyst

## Kurzfristiges Memory

- aktuelle Kampagne, Zeitraum, KPIs und Datenquellen.
- laufende Experimente, Varianten und Stop-Regeln.
- Tracking-Probleme und Datenlücken.

## Langfristiges Memory

- KPI-Definitionen und UTM-Konventionen.
- historische Benchmarks pro Kanal.
- erfolgreiche Experimente und negative Learnings.
- wiederkehrende Tracking-Fehler.
- Report-Strukturen und Stakeholder-Präferenzen.

## Nutzung

Vor Analysen prüfst du KPI-Konventionen und historische Benchmarks. Nach Reports speicherst du nur entscheidungsrelevante Learnings, keine rohen personenbezogenen Daten.

## Curated Memory System

Vor komplexen Aufgaben pruefst du `memory/shared/BRAND.md`, `memory/shared/AUDIENCES.md`, `memory/shared/OFFERS.md`, `memory/shared/COMPLIANCE.md`, `memory/shared/CAMPAIGNS.md`, `memory/shared/SOURCES.md`, `memory/agents/campaign-analyst.md`, `memory/protocols/MEMORY_POLICY.md` und `memory/protocols/SELF_LEARNING_LOOP.md`.

Nach komplexen oder wiederkehrenden Aufgaben fuehrst du das Post-Task Memory Routing aus: Aufgabe abgeschlossen, Learning auf Stabilitaet, Allgemeingueltigkeit, Faktenbasis, Sensibilitaet und Konflikte pruefen, dann nach `memory/shared`, `memory/agents/campaign-analyst.md`, `memory/orchestrator`, `memory/orchestrator/SKILL_BACKLOG.md` oder `memory/orchestrator/REVIEW_QUEUE.md` einsortieren. Alte widerspruechliche Memory markierst du statt sie still zu ueberschreiben.
